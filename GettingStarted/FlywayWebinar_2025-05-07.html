Redgate Flyway Advanced Features

Redgate Flyway is a powerful tool for managing database migrations. It provides a range of advanced features that can help you streamline your development process and ensure that your database schema is always in sync with your application code.

Some of the advanced features covered today are:

  • Static Data:
  • Code Analysis
  • Drift Detection
  • Change Reporting
Each of these is covered in more detail below.

Static Data

  • Also called lookup or reference data​
  • Static data is often used to populate checkboxes, drop downs, or radio buttons​
  • Values are usually the same in development and production​
  • Flyway can track tables with this data​
  • Flyway automatically detects changes​
  • Data changes are captured and stored in version control​
  • Changes are deployed along with other schema changes​

Code Analysis

Code analysis is a powerful feature of Redgate Flyway that allows you to analyze your database code for potential issues before you deploy it. This can help you catch bugs and performance issues early in the development process, saving you time and effort in the long run.

With code analysis, you can check for things like:

  • Unused objects
  • Missing indexes
  • Potential performance issues
  • Code style violations

    • Drift Detection

    Drift detection is another powerful feature of Redgate Flyway that allows you to detect changes to your database schema that have not been captured in your migration scripts. This can help you ensure that your database is always in sync with your application code, and can help you avoid issues caused by manual changes to the database.

    With drift detection, you can:

    • Detect changes to your database schema
    • Compare your database schema to your migration scripts
    • Generate reports on drifted objects

      • Change Reporting

      Change reporting is a feature of Redgate Flyway that allows you to generate reports on changes to your database schema. This can help you keep track of changes made to your database over time, and can help you identify potential issues before they become problems.

      With change reporting, you can:

      • Generate reports on changes to your database schema
      • Track changes made to your database over time
      • Identify potential issues before they become problems
